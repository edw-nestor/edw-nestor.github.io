How to install RethinkDB on Linux Mint 15 (Olivia)

RethinkDB’s official website has instructions for installing RethinkDB on Linux Mint 16 and Linux Mint 17, however if you are running on Linux Mint 15 (Olivia), those instructions just don’t work.

To install with binaries their instructions say to “add the RethinkDB repository to your list of repositories and install via apt-get:”

Linux Mint 16
1
2
 echo "deb http://download.rethinkdb.com/apt saucy main" | sudo tee
 /etc/apt/sources.list.d/rethinkdb.list
Linux Mint 17
1
2
echo "deb http://download.rethinkdb.com/apt trusty main" | sudo tee
/etc/apt/sources.list.d/rethinkdb.list
Install RethinkDB
1
2
3
wget -qO- http://download.rethinkdb.com/apt/pubkey.gpg | sudo apt-key add -
sudo apt-get update
sudo apt-get install rethinkdb

This does not work for Linux Mint 15 so I decided to try and compile from source.

To compile from source RethinkDB’s instructions say to download some essential libraries needed for the build.

Install build dependencies
1
2
3
4
sudo apt-get install build-essential protobuf-compiler python \
                     libprotobuf-dev libcurl4-openssl-dev \
                     libboost-all-dev libncurses5-dev \
                     libjemalloc-dev wget

This also did not work for Linux Mint 15 the command returned with the following error:

1
2
Unable to fetch some archives, maybe run apt-get update or try with
--fix-missing?

I ran apt-get update –fix-missing and still got the same error. At this point I thought I was out of luck until I decided to download the source code and just try to build it.

Get the source code
1
2
wget http://download.rethinkdb.com/dist/rethinkdb-1.16.2-1.tgz
tar xf rethinkdb-1.16.2-1.tgz
Build the server
1
2
3
4
cd rethinkdb-1.16.2-1
./configure --allow-fetch
make
sudo make install

That’s all there is to it. Enjoy RethinkDB.
